# CD4 Website Documentation

This document explains how the CD4 landing website is structured, where the entry points are, and which commands to run for development and production.

## 1) Project Identity

- Project type: Next.js landing website
- Product branding: `CD4`
- Framework mode: App Router (`src/app`)
- Language: TypeScript (strict)
- Styling: Tailwind CSS v4 + custom global CSS tokens

## 2) Prerequisites

- Node.js `>= 20` (defined in `package.json`)
- npm

If PowerShell blocks `npm`, use:

```powershell
npm.cmd install
npm.cmd run dev
```

## 3) Installation and Run Commands

Run from the landing project root:

```bash
npm install
```

Development server:

```bash
npm run dev
```

Production build:

```bash
npm run build
```

Run production server after build:

```bash
npm run start
```

Code quality:

```bash
npm run typecheck
npm run lint
```

## 4) Next.js Entry Points (Most Important)

### `src/app/layout.tsx`

- Root layout for entire app
- Global metadata/SEO title, description, OpenGraph, Twitter cards
- Wraps all routes

### `src/app/page.tsx`

- Home page entry
- Composes all landing sections:
  - Navbar
  - Hero
  - Features
  - Journey
  - Roles
  - Download
  - FAQ
  - Footer

### `src/app/globals.css`

- Global theme variables (brand colors, fonts, background gradients)
- Shared visual utilities and animations

### `src/app/sitemap.ts` and `src/app/robots.ts`

- Static SEO support routes generated by Next.js:
  - `/sitemap.xml`
  - `/robots.txt`

## 5) Content Architecture

### `src/content/landing.ts`

Single source of truth for editable marketing content:

- app name
- hero text
- feature list
- flow/journey blocks
- role sections
- FAQ
- default download links

### `src/lib/site-links.ts`

Resolves download links from environment variables.  
If env links are present, download cards automatically switch from `Coming Soon` to `Live`.

## 6) Environment Variables

Create `.env.local` from `.env.example`:

```bash
NEXT_PUBLIC_SITE_URL=https://cd4.app
NEXT_PUBLIC_ANDROID_STORE_URL=https://play.google.com/store/apps/details?id=com.cd4.app
NEXT_PUBLIC_IOS_STORE_URL=https://apps.apple.com/app/idYOUR_APP_ID
NEXT_PUBLIC_APK_URL=https://your-cdn.com/cd4-latest.apk
```

## 8) How the Website Was Built (Summary)

1. CD4 app modules were analyzed (AI triage, doctor booking, reports, subscriptions, doctor/admin flows).
2. Landing information architecture was mapped to those real modules.
3. Next.js project scaffolded with strict TypeScript and Tailwind.
4. UI split into reusable components for production-grade maintainability.
5. Content moved into data layer (`src/content/landing.ts`) for easy non-UI updates.
6. Download links made environment-driven to support fast release changes.
7. Typecheck, lint, and production build were validated.

## 9) Quick Update Checklist

If you need to update branding/copy quickly:

1. Edit `src/content/landing.ts`
2. Update logo image in `public/images/`
3. Update `.env.local` store/APK links
4. Run:

```bash
npm run typecheck
npm run lint
npm run build
```
